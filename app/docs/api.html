Products API

Clean, minimal, and ready-to-use REST API for managing products. Base path: /api/products

Authentication

This example project does not enforce authentication by default. If your app requires auth, add the appropriate headers (Bearer token) to requests.

Endpoints

GET /api/products

Returns paginated list of products.

QueryDescription
pagePage number (pagination)

Response (200)

{
  "current_page": 1,
  "data": [
    { "id": 1, "name": "Pen", "description": "...", "price": "4.50", "stock": 120 }
  ],
  "total": 20
}

curl

curl -s "{{base_url}}/api/products"
GET /api/products/{id}

Return a single product by ID.

Response (200)

{ "id": 1, "name": "Pen", "description": "...", "price": "4.50", "stock": 120 }

curl

curl -s "{{base_url}}/api/products/1"
POST /api/products

Create a new product. Content-Type: application/json

Request body

{
  "name": "String (required)",
  "description": "String (optional)",
  "price": "Number (required)",
  "stock": "Integer (required)"
}

Response (201)

{ "id": 21, "name": "...", "description": "...", "price": "10.00", "stock": 5 }

curl

curl -X POST "{{base_url}}/api/products" \
  -H "Content-Type: application/json" \
  -d '{"name":"Notebook","description":"A5 notebook","price":3.5,"stock":50}'
PUT /api/products/{id}

Replace product data completely. Provide all required fields.

Response (200)

{ "id": 1, "name": "Updated", "description": "...", "price": "5.00", "stock": 30 }
PATCH /api/products/{id}

Partial update: send any subset of writable fields.

DELETE /api/products/{id}

Delete the product. Response: 204 No Content on success.

Postman

Import the collection file docs/postman_collection.json into Postman. Set environment variable base_url (e.g. http://localhost).

Validation errors

If request data is invalid, API returns 422 Unprocessable Entity with an errors object.

{
  "message": "The given data was invalid.",
  "errors": {
    "name": ["The name field is required."]
  }
}

Notes

  • Replace {{base_url}} with your running application's base URL.
  • Routes are registered in routes/api.php.
  • Models and seeders are included so you can run php artisan migrate --seed when environment PHP version matches composer requirements.
